Chapter One

Introduction to Operating Systems

The operating system forms the invisible foundation upon which every computing experience is built — orchestrating hardware, software, and human interaction in perfect concert.

The term of an operating system (OS) is often used to describe a very complex piece of software which acts as a mediator between the computer's hardware and application programs. The OS is one of the fundamental types of system software and performs many critical tasks that enable a computer to function properly and allow users to use computer applications efficiently. An operating system is essentially a system program that controls and manages computer hardware resources while allowing user programs to run efficiently.

The importance of an operating system within computing is multifold. Firstly, by managing hardware resources such as the CPU, memory, storage devices, and input/output peripherals, an operating system ensures the effective utilization of these components. Rather than allowing applications to directly access hardware, which could lead to inefficiencies and conflicts, the OS abstracts this complexity. It allocates resources through scheduling algorithms and memory management techniques, ensuring that applications run smoothly and concurrently without disrupting one another.

Figure 1.1 — The Operating System as an intermediary layer between hardware and user applications.

Core Responsibilities of an OS

Operating systems manage hardware and software resources for a computer and ensure that these resources are used efficiently. Resource management refers to hardware as well as to software, and the services an OS provides to user applications are delivered via Application Programming Interfaces (APIs). An application uses a service of an operating system — for instance to manage processes — and the OS then manages the resources required by that process. For instance, the OS decides which part of the memory can be used by a process, and which part is already being used by another process.

Resource Management

Allocates CPU time, memory, and I/O devices efficiently across all running processes and applications.

Security & Protection

Provides authentication, access control, encryption, and process isolation to guard against threats.

User Interface

Presents graphical (GUI) or command-line (CLI) interfaces enabling humans to interact with the system.

Hardware Abstraction

Hides low-level hardware complexity through device drivers and system calls.

An operating system is not merely a program — it is the sovereign of every computation, arbitrating between the raw power of hardware and the aspirations of software.

There is also the importance of security in operating systems. Today all operating systems support various features that protect the operating system and the data on the computer from unauthorized access and damage by malicious programs. These can include things like user authentication, access control and data encryption. Operating systems also isolate processes and enforce permissions on file access. If the operating system is set up with adequate security policies then it can go a long way to protecting a computer from viruses and other security threats.

Furthermore, the user interface is of great importance when designing an Operating System. Not only do we interact with programs and files via the user interface — it is also the interface with the operating system itself. For this reason the user interface should be designed in such a way that the user is able to use the computer as easily as possible and complete tasks as quickly and efficiently as possible.

Chapter Two

OS Architecture: Layers & Components

The architecture of an operating system reveals how software layers cooperate to transform raw silicon into a programmable universe of boundless possibility.

The architecture of an operating system (OS) is an interrelated group of several layers or components of system software. These components function together to manage the various hardware or software resources of a computer, as well as to provide security and user interfaces to applications, and thus form the core of any computing system. An OS architecture can therefore be perceived as a number of interrelated components which together facilitate the efficient management of a variety of hardware or software resources.

Architectural Models

Monolithic Architecture

A monolithic architecture is a single large kernel that contains all core OS functions. This could include a memory manager, process scheduler, and device drivers — all running in the same address space. There is the potential for very fast communication between different parts of the kernel as they are using shared memory and are all part of the same process. However, this does create complexity as the kernel grows in size and more modules are added. A bug in one part of the kernel could cause problems for the entire OS, even resulting in a system crash with data loss.

Microkernel Architecture

A microkernel is the simplest form of an operating system. A small kernel performs the necessary services to the system while most other functions are performed by applications running in user space. Many device drivers, file systems, and networking protocols can run in user space. Their interaction with the kernel is by means of inter-process communication (IPC). This approach improves stability and modularity, though it may introduce performance overhead due to additional message-passing.

★ Key Concept: Separation of Concerns

The various layers of abstraction in an OS work to isolate and hide the lower levels of the system from the higher levels, as well as from the applications and users. This facilitates the separate, independent functions of the various components — enabling modular development, easier debugging, and improved security.

Another very important part of the OS architecture is the file system — organizing data into files and directories, managing permissions, and ensuring data integrity. The security layer spans all architectural levels, with authentication, access control, and data encryption woven throughout the system stack.

Chapter Three

The Kernel: Heart of the OS

Deep within every operating system lies the kernel — the sovereign core that translates human commands into electronic impulses, managing every resource with unrelenting precision.

The kernel, or core of the operating system (OS), is the central piece of software within a computer's OS that manages the system's resources by interacting directly with the computer's hardware while providing programs with a logical abstraction of the hardware. Thus the kernel is the heart of an operating system as it manages the services and resources made available to running application programs (Tanenbaum & Bos, 2015). The kernel is the lowest level software in the system and forms the basis upon which user space programs and the rest of the OS are built.

Figure 3.1 — The Kernel layout showing the boundary between user space and kernel space.

Kernel Responsibilities

Process Scheduling

The kernel schedules processes (i.e., it supports multitasking) and allocates the CPU time to processes on the basis of a scheduling algorithm. Most operating systems use preemptive scheduling — a process can be preempted by another at any time. Time slicing and priority scheduling are the two methods used by the kernel to manage CPU time. In this way, the kernel is able to manage the computer hardware efficiently.

Memory Management

The kernel uses virtual memory management to make it possible for multiple processes to be running on a computer at the same time without interference. Virtual memory allows the kernel to swap out parts of the physical memory to disk as needed, thus providing a computer with the ability to support many processes when physical memory is not sufficient.

System Call Handling

Another critical function is the management of system calls — the primary interface through which user applications communicate with the underlying hardware or request services from the kernel. By providing a defined set of system calls, the kernel abstracts the complexities of hardware interactions, presenting a simplified interface for application developers.

Figure 3.2 — User Mode vs. Kernel Mode Privilege Separation

Security Enforcement: Privilege Separation

In order to ensure the security of a system, the kernel enforces access control for users through a privilege separation model, which allows user mode and kernel mode. Applications running in user mode have only very limited access to the resources provided by the operating system. The kernel mode, on the other hand, is able to access all parts of the system hardware. This prevents malicious users from executing commands that could harm the system.

★ Kernel Sub-systems at a Glance

Process Management — creates, schedules, and terminates processes and threads.

Memory Management — virtual memory, paging, segmentation, MMU control.

I/O Management — device drivers, interrupt handling, buffering, caching.

File System — VFS layer, file metadata, permissions, disk allocation.

Security Sub-system — access control, cryptography services, audit logging.

Network Stack — TCP/IP implementation, socket management, firewall hooks.

User Interface: GUI & CLI

The user interface is the window through which human thought meets machine logic — shaping productivity, accessibility, and the very experience of computing itself.

The user interface is a critical component of an operating system and represents the way in which a user interacts with a system. The user interface of an operating system can take many different forms, broadly grouped into two main categories: graphical user interfaces and command-line interfaces. As outlined by O'Regan (2018), the user interface is an important component of an operating system, and in order to assist users in completing tasks, the interface must be designed to be efficient and support the goals of the user.

Figure 4.1 — Major operating systems, each with a distinctive user interface philosophy.

Graphical User Interface (GUI)

The graphical user interface, or GUI, is the more typical interface which makes use of icons, pictures, and windows on screen. The interface is generally used in conjunction with a pointing device and a screen. The user can use a mouse or even a touchscreen to click on items such as icons and buttons. The GUI greatly reduces the cognitive barrier for non-technical users, enabling them to perform complex operations through visual metaphors.

Command-Line Interface (CLI)

A command-line interface, or CLI, is a text-based interface generally used to access a variety of functions by entering commands and receiving results. CLIs are typically more efficient for experienced users or system administrators who need to complete specific tasks in a timely manner. The CLI provides the greatest degree of control and flexibility, and virtually any function can be automated with the use of scripts.

User Interface Security

User Interface security is another very important aspect, given the increasing misuse of computers by malicious individuals. The security of an Operating System interface is measured in terms of how easy it is for a user to use the interface securely. An OS interface should allow users to authenticate themselves, authorize access to system and application resources, and perform secure transactions. One of the biggest challenges an OS designer faces is striking a balance between interface security and usability.

Elements like consistency, responsiveness, and feedback mechanisms contribute to creating an environment where users can interact with the system fluidly. Hardware & Software Resource Management

Every CPU cycle, every byte of memory, every I/O transaction — the OS conducts this symphony of finite resources with meticulous, algorithmic precision.

One primary role of an operating system in hardware management is the orchestration of the CPU, memory, and input/output devices. The CPU, being the heart of computation, requires precise scheduling to maximize performance. Operating systems employ various scheduling algorithms, such as round-robin, priority scheduling, and shortest job first, to determine the sequence in which processes are allocated CPU time. These algorithms are critical, as they not only enhance responsiveness for interactive applications but also improve throughput for batch processing.

I/O Management

The OS manages peripherals or input/output devices by abstracting hardware interactions via device drivers, which act as translators between the OS and hardware devices. This architecture allows the OS to send commands to printers, disk drives, and network devices without applications needing to understand the intricate details of each device's operation. The OS employs techniques such as buffering and caching to optimize performance in data transfer operations.

Virtual Memory

Virtual memory allows programs to use more memory than is physically available on the computer. Parts of the program are stored on the hard disk and are loaded into physical memory as needed. This increases the range of applications that can be run, without being limited by physical memory. The OS creates the illusion of a virtually unlimited amount of memory for the user.

Chapter Six

Process & Memory Management

Every running program is a living process — born, scheduled, synchronised, and terminated in a perpetual dance orchestrated by the operating system kernel.

Process management is a fundamental responsibility of an operating system (OS) that focuses on the efficient handling and scheduling of processes. At its core, a process is a program in execution, and the management of these processes is crucial for optimizing system performance and ensuring user interactivity. The OS employs various strategies for multitasking, allowing multiple processes to appear as if they are executed simultaneously, despite the limitations of physical hardware.

Process States

Figure 6.1 — Process State Transition Diagram

There are five active states in which a process can be. The process could be in the new state, ready to be given some CPU time, running on the CPU, waiting for some I/O completion, or in the last state, terminated. Transitions between these states are taken based on several events. A process can enter the waiting state if it makes a system call that requires I/O. The process of switching between the different states is called a context switch.

Inter-Process Communication (IPC)

Process management also enables processes to communicate with each other in situations where several processes make use of the same resources. The various synchronization tools — for example semaphores, mutexes, and message passing — prevent race conditions where several processes access the same data at the same time.

Semaphores Integer counters for synchronization Mutual exclusion, resource counting High

Memory Protection & Allocation

Operating systems implement isolation mechanisms between processes to prevent them from inadvertently or maliciously accessing each other's memory spaces. This is critical in a multi-user or multi-tasking environment. Protection mechanisms often include hardware support, such as Memory Management Units (MMUs) that enforce access controls on memory regions using base and limit registers to define permissible access boundaries for each process.

Paging

Memory divided into fixed-size blocks (pages). OS maps virtual pages to physical frames. Eliminates external fragmentation.

Segmentation

Memory divided into variable-size segments (code, data, stack). Each segment has a base address and length.

Garbage Collection

Automatic reclamation of unreachable memory in managed languages. Prevents memory leaks and dangling pointers.

Buffer Overflow Protection

Stack canaries, ASLR, and DEP/NX bits prevent exploitation of memory boundary violations.

File System Management

The file system is the eternal librarian of computing — cataloguing, organizing, protecting, and serving data at the intersection of persistence and accessibility.

The management of files on a computer is another very important aspect of the Operating System. The file system provides an interface by which applications can store and retrieve files from the hard drives of a computer. The basic unit of information in a file system is a file — a block of computer storage that contains information such as text, images, or programs. Files are typically organized in a hierarchical tree-like structure, starting from the root directory.

File Security & Permissions

Security is a prominent concern within file system management. The operating system implements stringent access control mechanisms to regulate who can read, modify, or delete files. This is typically achieved through permission settings and user authentication, ensuring that sensitive data is safeguarded against unauthorized access. Through these security protocols, the file system functions not merely as a passive repository of data, but as an active participant in the protection of the integrity and confidentiality of the user's information.

Chapter Eight

Security in Operating Systems

In an era of escalating cyber threats, the operating system stands as the first and most critical line of defense — a sentinel guarding every bit, every process, every user.

The Operating System (OS) plays a crucial role as it acts as an intermediary between the user and the computer hardware. Given the fact that many programs run on the computer at any given time, the OS must ensure that each program uses the hardware in a safe and efficient manner. Thus, the Operating System plays a critical security role in protecting the computer system, any associated data, and other resources against unauthorized access, use, or harm.

Authentication & Authorization

User Authentication is the first line of defence. This process of verifying who users are is crucial to protecting their accounts and data from unauthorized access. Most operating systems today support several forms of authentication, including the typical login name and password. Since passwords are subject to many security risks, many systems now implement multi-factor authentication (MFA).

Memory Protection Process isolation via MMU, ASLR, DEP/NX Windows DEP, Linux ASLR
Encryption Data encryption at rest and in transit BitLocker, FileVault, TLS, dm-crypt

Common Security Vulnerabilities

Buffer Overflow

Exploits the difference between allocated buffer size and actual data input, potentially overwriting adjacent memory to execute arbitrary code.

Injection Attacks

Inserting untrusted data into commands (SQL injection, OS command injection, LDAP injection) that are then interpreted as instructions.

Privilege Escalation

Attacker gains root/administrator privileges by exploiting vulnerabilities in authentication or access control mechanisms.

Malware & Rootkits

Malicious software that infiltrates the OS, potentially hiding itself at the kernel level and persisting across reboots.

Modern operating systems take a multi-layered approach to security — because no single mechanism is ever sufficient against the full spectrum of contemporary threats.

Mitigation Strategies

To combat vulnerabilities, operating systems deploy a variety of mitigation strategies. One foundational strategy involves regular software updates and patch management. As developers identify vulnerabilities, timely updates can mitigate the risk of exploitation. The principle of least privilege is another key strategy — restricting users and applications to only those privileges necessary to complete their tasks. Secure coding practices, input validation, output encoding, and sandboxing complement these measures.

System Calls & APIs

System calls are the whispered requests of userland to the sovereign kernel — the formal language through which applications negotiate for the resources they need.

System calls are an integral mechanism that facilitates communication between user programs and the operating system (OS), functioning as the interface through which applications request services from the underlying OS. This operational framework is central to the architecture of modern operating systems. System calls serve as a kind of interface between users and the OS that allows a program to issue service requests using a fixed, pre-defined API and parameter list.

Application Programming Interfaces (APIs)

Application programming interfaces (APIs) of an operating system enable applications to communicate with the OS as well as with other applications. The APIs of an operating system make it possible for applications to access all resources of a computer while hiding the details of accessing hardware and managing resources. The design of APIs in an OS is often very systematic, with goals of increasing modularity, maintainability, and scalability.

★ The System Call Execution Flow

1. User Application calls a library function (e.g., printf())

2. C Library invokes the system call trap instruction (e.g., syscall on x86-64)

3. Mode Switch — CPU transitions from User Mode to Kernel Mode

4. Kernel validates parameters, performs the operation (e.g., writes to file descriptor)

5. Return — result returned to user space; mode switches back to User Mode

APIs also have a positive impact on the software development ecosystem, allowing for many innovative applications to be designed. Third-party developers can design their own applications which utilize the OS APIs efficiently, fostering a vibrant community of diverse software that runs on top of the operating system. By providing standardized interfaces, operating systems ensure portability across hardware generations and encourage broad software compatibility.

Specialized OS: RTOS, Distributed & Mobile

Beyond the desktop lies a vast ecosystem of specialized operating systems — each finely tuned for the demands of real-time response, distributed computing, or the constraints of mobile devices.

Real-Time Operating Systems (RTOS)

Real-Time Operating Systems (RTOS) are focused on meeting very high requirements with respect to real-time processing. These systems are required to deal with applications in which timely execution of events and processing of information in real time is a priority. They are used in a wide variety of applications ranging from car control systems to medical equipment and industrial automation processes.

Distributed Operating Systems

Distributed operating systems (DOS) manage a group of stand-alone computers in a network and treat them as if they formed a single supercomputer. A distributed operating system abstracts distributed computing, allowing users to use computers as if they were a single, monolithic system. The CAP theorem states that a distributed system can only fulfill two of three objectives: Consistency, Availability, and Partition tolerance.

Mobile Operating Systems

Mobile Operating Systems must manage hardware and software components while being extremely energy efficient and running on devices with limited processing power, memory, and storage. The two most widely used mobile OS are Android and iOS. Both have adopted micro-kernel-like architectures for their kernels. Mobile OSes also feature unique security models, such as Android's permission-based access control and iOS's sandboxing.

Android (Google)

Linux-based; open source; permission-based app model; runs on 70%+ of smartphones. Uses ART runtime.

iOS / iPadOS (Apple)

Unix-based (XNU kernel); closed ecosystem; strict sandboxing; tight hardware-software integration.

Virtualization & Cloud Computing

Virtualization shatters the physical constraints of hardware — one machine becomes many, one cloud becomes infinite, and the OS evolves into the conductor of a boundless orchestra.

Virtualization, the abstraction of hardware resources to support multiple operating environments, represents a new dimension in the evolution of operating system architecture. At the core of virtualization is the hypervisor or Virtual Machine Monitor (VMM). The hypervisor is a software layer that sits between the hardware and the guest operating systems (GOSs) running on top of it.

Figure 11.1 — Type 1 vs. Type 2 Hypervisors

Cloud Computing & OS Evolution

The rise of cloud computing has demanded an evolution in OS architecture, particularly in managing resources in virtualized environments. Traditional operating systems primarily focused on local hardware management. With cloud technologies, these systems need to extend their capabilities to handle distributed resources effectively. Virtualization enables multiple virtual environments to be created and run on a single physical server, with each virtual machine making use of hardware resources in a controlled and managed fashion.

Containerization

Containerization has become a paradigm shift in how applications are deployed, managed, and executed. Containers run on top of a host operating system, sharing that host's kernel but running in their own isolated user space. As such, containers are much more resource-efficient than virtual machines and can be started and stopped in a fraction of a second. Technologies like Docker and Kubernetes represent the current state of the art in container management.

Chapter Twelve

Case Studies: Windows, Linux & macOS

Three great operating systems — each a different philosophy of how computing ought to work, each a triumph of engineering and human creativity.

Microsoft Windows

The kernel of the Windows operating system is typically called the executive, which functions in two modes: user mode and kernel mode. User mode functions restrict usage of hardware by applications and services. Only system services function in kernel mode and can access hardware. Running in kernel mode alleviates the possibility of causing severe damage to the operating system and enhances system reliability.

★ Windows Architecture Highlights

Kernel (Executive) — Process/thread management, memory management, security reference monitor

Windows API — Abstracts hardware complexity; enables portability across Windows versions

User Account Control (UAC) — Limits standard user actions; requires elevation for sensitive operations

NTFS File System — ACL-based permissions, journaling, encryption (EFS), large volume support

GUI (Windows Shell) — Desktop, Start menu, taskbar; also supports PowerShell CLI

❧

Linux

Linux is an open-source operating system that was created by Linus Torvalds in 1991 and is currently developed by a global community. The architecture of Linux is based upon a monolithic kernel. The services provided by the OS are executed in kernel space. The monolithic architecture is efficient and fast because it does not require any inter-process communication between the different services of the OS.

A key feature of the Linux architecture is its customizability. The entire kernel may be compiled with only the components required for a given application — providing optimal use of available system resources. Linux supports a host of user interfaces including GNOME, KDE, XFCE, and many others, making it suitable for use from embedded systems to supercomputers.

❧

Apple macOS

The macOS operating system is based on Unix, providing performance and security of a Unix system. It is a multi-user system with excellent process management. The kernel is called XNU (X is Not Unix), which is a hybrid architecture combining Mach microkernel features with BSD subsystems. This architecture allows for high performance as well as good multitasking.

Emerging Trends & Future Directions

The operating system of tomorrow will be intelligent, adaptive, distributed, and deeply secure — shaped by AI, quantum hardware, and the relentless demands of a connected world.

The landscape of operating systems is changing at a breakneck speed, and they are constantly evolving to adapt to emerging trends in the field of information technologies, especially in the context of cloud computing, virtualization, security, and user experience. Today's operating systems are designed not only to support a variety of emerging information technologies but also to maximize user experience and satisfaction.

Key Emerging Trends

AI-Integrated OS

Operating systems incorporating machine learning for adaptive scheduling, predictive resource allocation, and real-time threat detection and response.

Zero-Trust Architecture

Every access request treated as a potential threat regardless of source. Security decisions made at every layer of the OS stack.

Cloud-Native OS

Operating systems designed from the ground up for cloud environments — distributing workloads across data centers with elastic scaling.

Natural Language UI

Conversational and voice-driven interfaces reducing barriers between human intent and computer action — making computing truly accessible.

Augmented Reality Integration

OS-level support for spatial computing, AR/VR overlays, and immersive interfaces blurring the line between physical and digital environments.

Open Source Expansion

Community-driven OS development gaining momentum, enabling greater customization, transparency, and collaborative security response.

The future operating system will not merely manage resources — it will intelligently anticipate needs, adapt to threats in real time, and evolve continuously through the contributions of a global community.

Power Management & Sustainability

Operating systems are increasingly incorporating sophisticated power management strategies. Dynamic Voltage and Frequency Scaling (DVFS) enables processors to run at different power levels as required by applications. When a system is under low load, processor cores can be powered down to save energy. When an application requires more processing power, the OS can raise the processing frequency. Energy-aware scheduling algorithms ensure full CPU utilization while minimizing wasted energy.

Multithreading & Concurrency

Multithreading and concurrency have become cornerstones for improved performance in current operating systems. Multithreading involves enabling a large number of threads of execution to run on available processor cores. By enabling these threads to run in parallel, the concept of multithreading in conjunction with concurrency helps tap the potential of modern multi-core hardware to improve overall system performance.

Updates, Patches & Continuous Evolution

It is critical to install all the latest updates for an operating system and apply all relevant patches to ensure a computing system remains secure and functional. An effective update management strategy that ensures timely installation of updates and patches is essential. Patches address security vulnerabilities, fix bugs, add new hardware support, and improve performance — making them a continuous and non-negotiable aspect of OS lifecycle management.
